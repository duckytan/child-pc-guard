# 验收标准（Acceptance Criteria）

**项目**：ChildPCGuard  
**文档编号**：AC-001  
**版本**：v1.0  
**日期**：2026-05-03

---

## 1. 概述

本文档定义各 Phase 交付物的完成定义（DoD，Definition of Done）。每个 Phase 的所有验收条件全部满足后，方可视为该 Phase 交付完成。

---

## 2. 通用 DoD（适用所有 Phase）

在任意 Phase 验收前，以下条件必须满足：

- [ ] 代码已合并到 `main` 分支，构建通过（`dotnet build` 无错误）
- [ ] 新增代码无 P0/P1 级别 Linter 警告
- [ ] 对应的单元测试已编写并通过
- [ ] `CHANGELOG.md` 已更新对应 Phase 的变更记录
- [ ] `task_plan.md` 对应 Phase 任务列表全部打勾

---

## 3. Phase 1 验收标准：MVP 时间管控

**目标**：部署后可实际使用，时间管控核心功能可靠运行。

### 功能验收

| 编号 | 验收条件 | 测试用例 |
|------|----------|----------|
| AC-1-01 | 以管理员身份运行 `install.ps1`，全程无报错，服务成功注册并启动 | 手动执行 |
| AC-1-02 | 将配额设为 2 分钟，使用电脑 2 分钟后，60 秒内触发锁屏 | FT-01 |
| AC-1-03 | 剩余配额 10/5/1 分钟时，分别弹出 Toast 通知 | FT-05~07 |
| AC-1-04 | 空闲 5 分钟（不操作鼠标键盘）时，计时暂停 | FT-08 |
| AC-1-05 | 将关机时间设为 +2 分钟，2 分钟后弹出关机倒计时并成功关机 | FT-09 |
| AC-1-06 | 重启电脑后，服务自动启动，今日已用时长从 state.json 正确恢复 | FT-10, RT-02 |
| AC-1-07 | 触发系统原生锁屏后，只有输入 Windows 登录密码方可解锁 | 手动验证 |

### 非功能验收

| 编号 | 验收条件 |
|------|----------|
| AC-1-NF-01 | 服务空闲状态下 CPU 占用 ≤ 0.5%，内存 ≤ 50 MB |
| AC-1-NF-02 | TimeTracker 单元测试覆盖率 ≥ 80%，全部通过 |
| AC-1-NF-03 | RuleEngine 单元测试覆盖率 ≥ 80%，全部通过 |

## 4. Phase 2 验收标准：防护加固

**目标**：标准用户（孩子）无法通过常规手段绕过或停止程序。

### 功能验收

| 编号 | 验收条件 | 测试用例 |
|------|----------|----------|
| AC-2-01 | 标准用户账户下，任务管理器无法结束 GuardService 进程 | PT-01 |
| AC-2-02 | 标准用户执行 `sc stop ChildPCGuard` 返回"拒绝访问" | PT-02 |
| AC-2-03 | 标准用户执行 `sc delete ChildPCGuard` 返回"拒绝访问" | PT-03 |
| AC-2-04 | 标准用户无法删除或修改安装目录中的任何文件 | PT-04 |
| AC-2-05 | 标准用户无法打开或修改 `config.bin` | PT-05 |
| AC-2-06 | 标准用户无法修改服务相关注册表键 | PT-06 |
| AC-2-07 | 配置文件使用 AES-256 加密，以十六进制编辑器打开无法读取明文 | 手动验证 |
| AC-2-08 | 检测到 NTP 时间偏差 > 5 分钟时，触发锁屏并在日志中记录 | 手动验证 |
| AC-2-09 | 安全模式启动时，程序触发关机（在 VM 环境中验证） | 手动验证 |

### 渗透测试通过要求

PT-01 ~ PT-09 全部通过（无一失败）。

## 5. Phase 3 验收标准：自定义锁屏

**目标**：自定义锁屏覆盖层替换系统原生锁屏，牢不可破。

### 功能验收

| 编号 | 验收条件 | 测试用例 |
|------|----------|----------|
| AC-3-01 | 锁屏界面覆盖全部显示器（单显 + 双显均测试），无可见缝隙 | FT-01 |
| AC-3-02 | 锁屏期间 Alt+Tab、Win+D、Win+L、Win+Tab 等快捷键无效 | PT-10, PT-11 |
| AC-3-03 | 输入正确密码后锁屏消失，桌面恢复正常 | FT-02 |
| AC-3-04 | 连续 3 次错误密码后，密码框锁定 5 分钟，显示倒计时 | FT-03 |
| AC-3-05 | 5 分钟锁定期满后，密码框自动恢复可用 | FT-04 |
| AC-3-06 | LockOverlay 进程被杀死后，GuardService 在 30 秒内重新触发锁屏 | RT-03 |
| AC-3-07 | 隐藏快捷键（Ctrl+Alt+Shift+F12 ×5）可触发紧急解锁流程 | 手动验证 |

## 6. Phase 4 验收标准：双进程互保

**目标**：AgentA 和 AgentB 互相守护，大幅提升抗杀能力。

### 功能验收

| 编号 | 验收条件 | 测试用例 |
|------|----------|----------|
| AC-4-01 | 安装后 AgentA、AgentB 均以 SYSTEM 权限运行 | 手动验证（任务管理器） |
| AC-4-02 | 以管理员强制结束 AgentA 进程，30 秒内 AgentB 自动重启 AgentA | IT-04 |
| AC-4-03 | 以管理员强制结束 AgentB 进程，30 秒内 AgentA 自动重启 AgentB | 对称测试 |
| AC-4-04 | 标准用户账户下无法结束 AgentA 或 AgentB 进程 | PT-08, PT-09 |
| AC-4-05 | 进程名在任务管理器中显示为伪装名（非 AgentA/AgentB） | 手动验证 |
