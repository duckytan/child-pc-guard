# 需求规格说明书（SRS）

**项目**：ChildPCGuard  
**文档编号**：SRS-001  
**版本**：v1.0  
**日期**：2026-05-03  
**状态**：已基线

---

## 1. 文档目的

本文档描述 ChildPCGuard 的全部功能需求和非功能需求，作为开发、测试、验收的共同基准。本文档与 `ChildPCGuard_项目方案.md` 配合使用，前者为需求基线，后者为技术实现方案。

---

## 2. 术语与缩写

| 术语 | 说明 |
|------|------|
| 家长 | 管理方，持有管理员密码，使用 AdminPanel |
| 孩子 | 被管控方，使用标准用户账户 |
| 时间配额 | 每日允许使用电脑的最长时长 |
| 锁屏 | 程序触发的自定义覆盖层，非 Windows 原生锁屏 |
| 服务 | GuardService，以 SYSTEM 权限运行的 Windows 服务 |
| Agent | AgentA / AgentB，以 SYSTEM 权限运行的双进程互保守卫 |

---

## 3. 干系人与用户角色

| 角色 | 描述 | 主要需求 |
|------|------|----------|
| 家长 | 安装、配置程序，查看孩子使用记录 | 简单易用的管理界面；规则灵活可调 |
| 孩子 | 正常使用电脑，受时间管控 | 锁屏前有明确提醒；密码错误有保护不被长期封锁 |
| 开发者 | 构建、测试、维护程序 | 清晰的模块边界；可测试的接口 |

---

## 4. 功能需求

### 4.1 时间管控（FR-01 ~ FR-07）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-01 | 系统应按工作日 / 周末分别记录孩子的电脑使用时长 | 必须（M） |
| FR-02 | 家长可分别为工作日和周末设置每日时间配额（分钟，1 ~ 1440） | 必须（M） |
| FR-03 | 鼠标/键盘空闲超过 5 分钟（可配置）时，暂停计时 | 必须（M） |
| FR-04 | 配额剩余 10 分钟、5 分钟、1 分钟时，分别弹出 Toast 通知 | 必须（M） |
| FR-05 | 配额用完后，系统应在 60 秒内触发锁屏 | 必须（M） |
| FR-06 | 重启后，程序应自动读取今日已用时长并继续计时 | 必须（M） |
| FR-07 | 家长可通过 AdminPanel 为今日追加额外时长（0 ~ 480 分钟） | 应该（S） |

### 4.2 锁屏（FR-08 ~ FR-14）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-08 | 锁屏界面应覆盖全部显示器，不留任何可操作区域 | 必须（M） |
| FR-09 | 锁屏期间，所有快捷键（Alt+Tab、Win、Ctrl+Alt+Del 除外）应被拦截 | 必须（M） |
| FR-10 | 输入正确管理密码后，锁屏界面解除 | 必须（M） |
| FR-11 | 连续输入错误密码 3 次后，密码输入框锁定 5 分钟，显示倒计时 | 必须（M） |
| FR-12 | 锁屏界面应显示：剩余锁定时间（若被锁定）、今日已用时长 | 应该（S） |
| FR-13 | 紧急情况下，家长可通过隐藏快捷键（Ctrl+Alt+Shift+F12 ×5）触发特殊解锁流程 | 应该（S） |
| FR-14 | 锁屏进程被意外终止后，GuardService 应在 30 秒内重新触发锁屏 | 必须（M） |

### 4.3 自动关机（FR-15 ~ FR-18）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-15 | 家长可配置每天自动关机时间（精确到分钟） | 必须（M） |
| FR-16 | 关机前 60 秒，系统应显示关机倒计时通知 | 必须（M） |
| FR-17 | 关机由双重机制保障：GuardService 计时器 + Windows 任务计划程序 | 必须（M） |
| FR-18 | 家长可通过 AdminPanel 取消当日自动关机（仅当日有效） | 应该（S） |

### 4.4 管理员界面 AdminPanel（FR-19 ~ FR-26）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-19 | AdminPanel 启动时需要输入管理密码（BCrypt 哈希验证） | 必须（M） |
| FR-20 | 主界面显示：今日使用时长、剩余配额、服务运行状态 | 必须（M） |
| FR-21 | 规则页可修改：工作日/周末配额、关机时间、空闲阈值 | 必须（M） |
| FR-22 | 规则修改保存后，GuardService 应在 30 秒内使用新规则 | 必须（M） |
| FR-23 | 临时操作：追加时间、暂停管控（最长 30 分钟）、立即锁屏 | 应该（S） |
| FR-24 | 日志页显示近 30 天每日使用时长记录，可按周查看 | 应该（S） |
| FR-25 | 管理密码可在 AdminPanel 内修改（需输入旧密码） | 应该（S） |
| FR-26 | 提供注册全局热键（默认 Ctrl+Alt+Shift+G）快速呼出 AdminPanel | 可选（C） |

### 4.5 自我保护（FR-27 ~ FR-34）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-27 | 标准用户（孩子账户）不得通过任务管理器终止 GuardService 进程 | 必须（M） |
| FR-28 | 标准用户不得通过 `sc stop` 停止 GuardService | 必须（M） |
| FR-29 | 安装目录的文件不得被标准用户修改或删除 | 必须（M） |
| FR-30 | 配置文件使用 AES-256 加密存储，明文不落盘 | 必须（M） |
| FR-31 | AgentA 与 AgentB 互相监控，任一死亡后另一应在 30 秒内重启对方 | 必须（M） |
| FR-32 | 程序检测到系统时间与 NTP 偏差超过 5 分钟时，触发锁屏并记录日志 | 应该（S） |
| FR-33 | 检测到以安全模式启动时，程序应立即触发关机 | 应该（S） |
| FR-34 | 服务注册表键受 DACL 保护，标准用户不得修改 | 必须（M） |

### 4.6 安装与卸载（FR-35 ~ FR-39）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-35 | 提供 PowerShell 安装脚本，引导家长完成安装 | 必须（M） |
| FR-36 | 安装脚本支持 `-Silent` 参数进行无交互安装 | 应该（S） |
| FR-37 | 安装脚本自动检测 .NET 8 运行时，缺失时给出安装提示 | 必须（M） |
| FR-38 | 卸载需要输入管理密码，验证通过后方可执行 | 必须（M） |
| FR-39 | 卸载后清除所有服务注册、任务计划、ACL 保护和配置文件 | 必须（M） |

## 5. 非功能需求

### 5.1 性能需求

| 编号 | 需求 | 指标 |
|------|------|------|
| NFR-01 | GuardService 常驻内存占用 | ≤ 50 MB |
| NFR-02 | AgentA / AgentB 单个内存占用 | ≤ 20 MB |
| NFR-03 | CPU 占用（空闲状态） | ≤ 0.5%（测量周期 60 秒） |
| NFR-04 | 从检测到配额用尽到触发锁屏的延迟 | ≤ 60 秒 |
| NFR-05 | AgentA 死亡后 AgentB 检测并重启的时间 | ≤ 30 秒 |

### 5.2 可靠性需求

| 编号 | 需求 |
|------|------|
| NFR-06 | 服务崩溃后，SCM 自动重启，重启后读取持久化状态继续运行 |
| NFR-07 | 锁屏触发后，即使 AdminPanel 进程崩溃，锁屏不得自动解除 |
| NFR-08 | 正常使用场景下，服务运行时间 MTBF ≥ 30 天 |

### 5.3 安全需求

| 编号 | 需求 |
|------|------|
| NFR-09 | 管理密码使用 BCrypt（cost factor ≥ 12）哈希存储 |
| NFR-10 | 配置文件中的敏感字段（密码哈希、规则）使用 AES-256-GCM 加密 |
| NFR-11 | IPC 命名管道的访问 ACL 仅允许 SYSTEM 账户和管理员组 |
| NFR-12 | 日志文件不得记录管理密码明文或哈希 |

### 5.4 兼容性需求

| 编号 | 需求 |
|------|------|
| NFR-13 | 支持 Windows 10 version 1903 及以上 |
| NFR-14 | 支持 Windows 11 所有正式版本 |
| NFR-15 | 支持单显示器和多显示器配置 |
| NFR-16 | 不依赖第三方云服务或网络账号，完全本地运行 |

### 5.5 可维护性需求

| 编号 | 需求 |
|------|------|
| NFR-17 | 单元测试覆盖率：TimeTracker、RuleEngine ≥ 80% |
| NFR-18 | 所有模块通过接口解耦，支持单元测试中的依赖注入 |
| NFR-19 | 日志级别可配置（Debug / Info / Warning / Error） |

---

## 6. 约束条件

- 目标机器必须安装 .NET 8 Desktop Runtime
- 孩子账户必须是 Windows 标准用户（非管理员）
- 安装必须在管理员账户下执行
- 程序不依赖网络连接（NTP 校验为可选增强功能）
