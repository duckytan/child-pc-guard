| [vlytvynchyk/pc-usage-timer](https://github.com/vlytvynchyk/PC-Usage-Timer) | C# / .NET 9 | 0 | 倒计时锁屏，PIN 保护，内嵌 Web 服务远程控制，键盘钩子拦截快捷键 | ✅ 值得参考 |

### 1.2 关键调研发现

**发现 1：Windows 家长控制开源项目极度匮乏**
- GitHub 上 `parental-control` topic 共 184 个仓库，但针对 Windows 且具备自我保护能力的**几乎为零**
- 多数项目功能简单，缺乏防篡改机制，对有电脑基础的孩子形同虚设

**发现 2：ScreenTimeGuard 是最值得借鉴的参考项目**
该项目采用以下多层防护体系：
```
普通用户攻击向量 → 防护结果：
[sc stop]          → ❌ 服务 DACL 拒绝
[任务管理器 Kill]  → ❌ SYSTEM 进程，需 SeDebugPrivilege
[删除程序文件]     → ❌ 目录 ACL 仅 SYSTEM/Admins 可访问
[修改系统时间]     → ❌ UTC 时区锁定
[重启电脑]         → ❌ 开机自启（auto-start）
[程序意外崩溃]     → ❌ SCM 3次失败自动重启
```

**发现 3：商业软件现成解决方案**
如不想自己开发，以下商业/官方方案可直接使用：
- **Microsoft Family Safety（微软家庭安全）**：免费，与 Windows 深度集成，支持屏幕时间、内容过滤，但需联网且孩子知道是家长控制
- **Qustodio**：商业软件，功能强大，有中文版
- **Net Nanny**：老牌商业家长控制软件
- **Norton Family**：诺顿家庭安全

## 第二部分：需求分析

### 2.1 核心需求

| 需求编号 | 需求描述 | 优先级 |
|----------|----------|--------|
| R-01 | 每天设定可玩电脑的总时长（例如工作日 2 小时，周末 4 小时） | P0 |
| R-02 | 超过时间自动锁屏，必须输入家长密码才能解锁 | P0 |
| R-03 | 每天晚上 22:00 自动关机 | P0 |
| R-04 | 程序以隐秘后台服务方式运行，不显示在任务栏 | P0 |
| R-05 | 普通用户（孩子账户）无法通过任务管理器结束进程 | P0 |
| R-06 | 普通用户无法卸载或停止程序 | P0 |
| R-07 | 意外崩溃或系统重启后自动恢复 | P0 |
| R-08 | 家长可通过密码访问管理界面修改规则 | P1 |
| R-09 | 记录使用日志（每天玩了多久） | P1 |
| R-10 | 锁屏前提前 10/5/1 分钟弹出倒计时提醒 | P1 |
| R-11 | 防止修改系统时间绕过 | P1 |
| R-12 | 锁屏界面拦截 Alt+Tab、Win 键等快捷键 | P1 |

### 2.2 威胁模型（孩子的反制手段）

作为用了 5 年电脑、会任务管理器的初中生，其可能的反制手段：

| 攻击手段 | 危险等级 | 应对方案 |
|----------|----------|----------|
| 任务管理器结束进程 | 高 | 服务以 SYSTEM 权限运行，普通用户无法 Kill |
| sc stop / net stop | 高 | 服务 DACL 配置拒绝非管理员操作 |
| 删除程序文件 | 中 | 安装目录 ACL 拒绝标准用户写入 |
| 修改系统时间 | 中 | 使用网络时间（NTP）校验，UTC锁定策略 |
| 重启电脑 | 中 | 开机自启，重启后自动恢复计时 |
| 注册表修改启动项 | 中 | 注册表 ACL 保护 |
| 以管理员运行 cmd | 高 | **关键限制**：孩子账户不得有管理员权限 |
| 安全模式启动 | 高 | 可在安全模式下也注册服务（高级方案） |
| 搜索反制工具 | 中 | 可选：屏蔽特定网站、进程白名单 |

### 3.3 模块详细设计

#### 模块 1：GuardService（守护服务）

**注册为 Windows 服务，以 LocalSystem 权限运行**

```csharp
// 核心功能循环（每5秒执行一次）
while (!stoppingToken.IsCancellationRequested) {
    1. 检查当前时间是否在允许时段内
    2. 累计今日使用时长
    3. 判断是否需要锁屏（超出时长或超出时段）
    4. 判断是否到达自动关机时间（每天 22:00）
    5. 发送剩余时间通知（10/5/1 分钟预警）
    6. 持久化 state.json
    await Task.Delay(5000, stoppingToken);
}
```

**计时逻辑**：
- 计时从用户登录（会话开始）计起
- 状态持久化到 `state.json`，重启后恢复累计时长
- 支持"暂停时间"（孩子去吃饭时家长可暂停）

#### 模块 2：自我保护机制

**层次 1：Windows 服务配置**
```powershell
# 注册服务：LocalSystem 权限，开机自启
sc.exe create ChildPCGuard binPath="C:\Program Files\ChildPCGuard\GuardService.exe" 
    start=auto obj=LocalSystem DisplayName="Windows Security Update Service"

# 失败自动重启（3次，每次间隔1秒）
sc.exe failure ChildPCGuard reset=86400 actions=restart/1000/restart/1000/restart/1000
sc.exe failureflag ChildPCGuard 1
```

> **伪装技巧**：服务显示名称设为 "Windows Security Update Service"（让孩子以为是系统服务）

**层次 2：服务 DACL 保护**
```powershell
# 设置服务 DACL，拒绝非管理员用户停止/查询服务
# SDDL 字符串：只有 SYSTEM 和 Administrators 可以控制服务
$sddl = "D:(A;;CCLCSWRPWPDTLOCRRC;;;SY)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;BA)"
sc.exe sdset ChildPCGuard $sddl
```

**层次 3：进程 ACL 保护**
```csharp
// 服务启动时设置自身进程 DACL，拒绝普通用户 PROCESS_TERMINATE
private static void ProtectProcess() {
    // 创建只允许 SYSTEM 和 Administrators 访问的安全描述符
    // 普通用户无法获取进程句柄来终止它
    SetProcessAcl(GetCurrentProcess());
}
```

**层次 4：文件系统保护**
```powershell
# 安装目录 ACL：只有 SYSTEM 和 Administrators 有完全控制
$acl = Get-Acl "C:\Program Files\ChildPCGuard"
$acl.SetAccessRuleProtection($true, $false)  # 禁用继承
# 移除 Users 组权限，只保留 SYSTEM + Admins
Set-Acl "C:\Program Files\ChildPCGuard" $acl
```

**层次 5：注册表保护**
```powershell
# 保护启动注册表项不被修改
# HKLM\SYSTEM\CurrentControlSet\Services\ChildPCGuard 设置 ACL
```

**层次 6：双进程互保（可选，高级）**
```
GuardService.exe ←──→ WatchDog.exe
     互相监控对方是否存活
     任意一个被终止，另一个立即重新启动它
```

// 方案1：服务内部定时器

## 第五部分：关键注意事项

### 5.1 最重要的前提条件

**将孩子的 Windows 账户设置为"标准用户"（非管理员）**

这是整个方案的基石。如果孩子的账户有管理员权限，所有软件保护都可能被绕过。

```
控制面板 → 用户账户 → 管理其他账户 → 选择孩子账户 → 更改账户类型 → 标准
```

### 5.2 无法防御的情形

| 场景 | 说明 |
|------|------|
| 孩子知道管理员账户密码 | 一旦知道密码，可以切换到管理员账户操作 |
| 安全模式启动（不加安全模式保护时） | Win10/11 可在恢复环境中启动安全模式 |
| 重装系统 | 极端情况，需要在 BIOS 中禁用 USB 启动 |
| PE 系统启动 | 同上，需 BIOS 密码 |

**建议额外措施**：
- 设置 BIOS/UEFI 密码，禁用 USB 启动
- 家长账户密码不告诉孩子
- 对孩子说明规则，培养自律（软件是辅助，不是替代）

### 5.3 亲子关系建议

建议以**透明方式**告知孩子程序的存在，只是不告诉其技术细节和密码。这样：
- 孩子知道有规则，心理上更容易接受
- 避免被发现后产生严重的信任危机
- 可以和孩子协商规则（如考试前可以暂停，假期适当放宽）
